FIX SCONTO ORDINI

1. Sostituisci nel progetto i due file mantenendo i percorsi:
   src/modules/orders/pages/NewOrder.jsx
   src/modules/orders/services/priceEngine.js

2. In Supabase > SQL Editor esegui per intero:
   supabase/migrations/20260718235500_fix_categorie_sconto_ordini.sql

3. Arresta Vite con Ctrl+C e riavvia:
   npm run dev

4. Nel browser esegui Ctrl+F5.

5. Apri Nuovo ordine. In alto comparirà il conteggio delle regole caricate.
   Deve indicare almeno: matrice 500, particolarità 64, pagamenti 1.

6. Il cliente dello screenshot deve mostrare Categoria sconto 2, non 0.
   La riga IT0001 mostrerà anche la categoria articolo effettivamente letta.

7. Per controllare i dati esegui docs/VERIFICA_FIX_SCONTO.sql.

Git non serve per il test locale. Dopo la verifica positiva:
   git add .
   git commit -m "Fix categorie e calcolo sconti ordini"
   git push origin main
